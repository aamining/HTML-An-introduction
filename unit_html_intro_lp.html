  2. Understand Web Development Technologies and Tools

History of the Web

  1. Explain the Evolution of the Web
  2. Explain the Internet Archive Wayback Machine

Web Development Technologies and Tools

  1. Networking Models & Layers (OSI & TCP/IP Models & Error Codes)
    2. Ask students to:
    • Go through Intro to WWW Slides

      • Note: Topics include: Intro to WWW (Networks and Host Nodes), Server Types (including DNS), IP Addresses, Domain Names, Network Architecture (Client Server Model), Peer-to-peer (P2P) Architecture, Intranet vs Internet vs Extranet, Cloud Computing (IaaS, Paas, SaaS), Hypertext Documents, HTTP Messages & Method Types, Web Browsers and Web Servers, Web Application Architecture (Request/Response), Web Caches (Proxy Servers), HTTP Response Status Codes, TCP/IP, Standard Protocol Architectures, OSI Layered Reference Model (Standard Framework), HTML exchange request using HTTP, OSI vs TCP/IP, TCP/IP Protocol Graph, TCP/IP Boundaries

    • Explain HTTP Status Codes
    • Read List of OSI Application Layer Protocols (HTTP, SMTP, POP, FTP)
  2. HTML
    3. Ask students to:
    • Read Mozilla Introduction to HTML
    • Create and preview a basic HTML page HTML5 Intro
  3. DOM, Global Objects (Window), Events
    4. Ask students to:
    • Script Tag
    • Document OnLoad Event
  4. Character Encodings, Document Types, Meta Tags
    5. HTML Markup Validator (Linter)
  5. Browsers & Responsive Design & Refresh
    6. Ask students to:
    • Read Browser Detection Sniffers with User Agent and Incompatibility Examples
    • Apply the Browser Detection Sample Code using Chrome Dev Tools Console
    • Try using Chrome Dev Tools Emulator. Open the Chrome Developer Tools (CMD+OPTION+I). Click the Toggle Device Toolbar icon (CMD+SHIFT+M). View Previews of a Webpage using difference Mobile and Desktop devices
    • Try Browserstack for Cross-Browser Compatibility Testing
  6. Polyfills vs Libraries (Fix Gaps in Functionality such as support for older browsers vs Improve Existing Functionality)
    7. Briefly discuss the difference
  7. Privacy with SSH (OSI Application Layer)
    8. Ask students to:
    • Show how we will use SSH to Connect to Github without supplying username/password each time from Command Line
  8. Sessions (OSI Session Layer)
  9. Security with SSL Certificates & Protocols, TLS (OSI Presentation/Session Layer), HTTPS
    10. Ask students to:
    • Read Intro to SSL Certificates & Protocols, TLS (OSI Presentation/Session Layer), HTTPS
    • Show how Let's Encrypt (free, automated, and Open Certificate Authority may be used for HTTPS on a Website, and how Heroku uses it for Automated Certificate Management
  10. Security with Content Security Policies (CSP) to mitigate Cross-Site Scripting (XSS)
    11. Ask students to:
    • Evaluate the strength of a CSP using this Google CSP Evaluator
    • Implement Lockdown CSP Example by adding between the HTML Tags of the HTML file and then refresh the Webpage in the browser and view the Console in Chrome Dev Tools, which shows error Refused to apply inline style because it violates the following Content Security Policy directive: "default-src 'none'". Either the 'unsafe-inline' keyword, .... Note also that 'style-src' was not explicitly set, so 'default-src' is used as a fallback.
    • Challenge: Fix the error to allow inline Script and Style Tags over HTTPS using CSP Directives Solution: Add
  11. Anonymity with HTTP & SOCKS Proxies (OSI Application Layer)
  12. Web Crawlers, Robots File, and SEO
  13. Debugging (Chrome Dev Tools)
    14. Show students a simple example:
    • Open Chrome Dev Tools. Go to "Sources" Tab. Create a New Snippet of Code: Sources > Snippets > New Snippet var username = "Student" var welcomeMessage = `hello ${username}`; console.log(welcomeMessage);
    • Add a Breakpoint on the 2nd Line of code
    • Run the code. View the Console output. Hover over welcomeMessage after breakpoint is triggered. Enter welcomeMessage in the Console to show that the value is stored in the variable of the Window Global scope window.welcomeMessage. Inspect other variables in the Global scope in the right-hand side panel
    Ask students to:
    • Complete the Discover Dev Tools course
    • Read the Chrome Dev Tools documentation
  14. HTML 5 Canvas & Touch Event Listeners and Handlers for Tracking and Logs
    15. Ask students to:
    • Read Multi-touch Web Development Guide
    • Read Web API Docs on Touch Events
    • Download unit_html_intro/touch_events.html. Run a local Ruby web server from the same directory to host files in that folder on port 8000 with: ruby -run -e httpd . -p 8000. Go to http://localhost:8000/unit_html_intro/touch_events.html in your browser. Open the Chrome Developer Tools (CMD+OPTION+I). Click the Toggle Device Toolbar icon (CMD+SHIFT+M). Click the "Initialise" Button in the DOM to setup the touch event listeners. Emulate Touch Events in the HTML 5 Canvas by holding down the mouse and draging it around.
    • Load your website using your web server using your IP Address from another students computer. Find out your Public IP Address with ifconfig | grep inet and run the website on another students computer. Open and Emulute Touch Events using the server on your computer but running it from their browser http://:8000/unit_html_intro/touch_events.html
    • Load your website using your web server using a Domain Name generated by Ngrok with ngrok http 80, which tunnels traffic from GET requests made to the domain name that they generate through to your local HTTP server
    • Read the Vim Tutorial vimtutorial
    • Edit the HTML file using Vim and observe changes or bugs produced
  15. HTML Drag & Drop API
    16. Ask students to:
    • Read Drag and Drop Guide (drag events inherit from mouse events)
    • Challenge: Implement Drag & Drop using similar approach to the Touch Event example
    • Challenge: Implement File Drag & Drop using similar approach to Touch Event example
  16. DNS / IP Addresses, Request/Response
    17. Ask students to:
    • Show diagram of how DNS Server's map to IP address where files are hosted
    • Find the DNS information of a Domain Name in Terminal or in Web Browser whois heroku.com
    • Show how Whois works man whois
    • Trace the Route to an Host Name or given IP address with traceroute google.com, which shows the routers your packets travel through and how long it takes between each router. If the routers have DNS entries, traceroute will list the router names and their geographic location.
  17. Web Standards
    18. WWW Organisations (W3C Standards)
  18. Web Accessibility Guidelines
    19. Ask students to:
    • Read about Web Accessibility
    • Identify Before and After usage of Web Accessibility on a Website
    • Update a HTML Website to use ARIA Tags
    • Verify that ARIA Tags (i.e. alt tag used )applied to images are read by ChromeVox Screen Reader
  19. CSS Animation & 3D Transforms
    20. Ask students to:
    • Read Intro to CSS Animations
    • Show demos of CSS Animations with Keyframes and Browser Compatibility
  20. Browser Local Storage, Databases, Caches, Cookies
    21. Ask students to:
    • Read about Browser Local Storage, Databases, Caches, Cookies
    • Show demos of: Set, Get, Remove Session Storage Setting a Cookie Setting Local Storage items
  21. Offline Web App Cache
    22. Ask students to:
    • Read about Offline Web App Cache
    • Try to implement: Offline Web App Cache
  22. Web Workers

Guided Practice - Exercises In-Class (Canvas)

  1. Re-Design Process for an existing Website from Internet Archive Wayback Machine (IAWM)
    • Choose a website
    • Compare the original and modern versions of the website
    • Write down the technologies and design techniques they changed and why
    • Choose a different website from IAWM to re-design yourself
    • Identify users of the site (target market)
    • List features your new site will need
    • Sketch the re-design on butchers paper
    • Create wireframe using software (i.e. Axure, Figma, Balsamiq, etc)
    • Research colours palette and typography to satisfy users (independent of product being sold)
    • Create mood board (i.e. with Pinterest)
    • Choose brand asset colours and typography based on your research and mood board
    • Explain justification for decisions make in the design process
